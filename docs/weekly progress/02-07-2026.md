# Progress — 02 Juli 2026

## Domain 3: Integrasi Inventori & Stock Control dengan Input Harian

Menyambungkan **Konsumsi pakan** dan **Pengobatan** ke stok fisik, menambah
auto-increment stok **telur** dari panen, dan membangun modul admin Inventori.

### Skema & migrasi
- `InventoryStock` → `@@unique([item_id, location_id])`.
- `MedicalRecord` → tambah `item_id` (nullable, `onDelete: SetNull`) + `quantity_used`.
  `medicine_name` tetap untuk fallback teks bebas.
- Migrasi `20260702000000_add_inventory_integration_fields` diterapkan.
- Seed: item Obat (Amoksisilin), Vitamin (Vitamin B Kompleks), Telur (saldo 0),
  Lainnya (Solar) + baris `InventoryStock` awal & `min_stock_alert` untuk pakan.

### Keputusan desain
- **Tipe mutasi = konstanta string** (bukan enum Prisma) di
  `features/inventory/lib/stock-mutation-types.ts` — hindari migrasi enum yang
  breaking, cocok dengan `mutation_type` yang sudah free-text. Prefix `IN_`/`OUT_`
  menentukan arah. Tersedia `OUT_VACCINE`/`OUT_SALES` untuk modul mendatang.
- **`MedicalRecord.item_id` → `SetNull`**: hapus item tidak menghapus riwayat.
- **Stok negatif ditolak**: OUT memakai `updateMany(quantity >= qty)` (atomik,
  anti-oversell tanpa row lock). Lewat ambang batas → sukses + peringatan; kurang
  stok → gagal dengan pesan Indonesia. `allowNegative` hanya untuk rekonsiliasi.
