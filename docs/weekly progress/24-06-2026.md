## Yang sudah dikerjakan (minggu-minggu terakhir)

### Arsitektur & fondasi

- Pivot input lapangan dari PWA Next.js ke **Expo React Native** (`docs/ecosystem.md`).
- Proteksi rute `proxy.ts`; CORS untuk dev Expo; envelope API standar.
- Seed RBAC, tenant, user staff (`staff.kandang`) untuk uji lapangan.

### Domain 1 — hampir selesai

- Better Auth + session multi-tenant (`active_tenant_id`).
- CRUD tenant, pengguna, peran & permission; branding tenant (logo).
- Mobile auth endpoint staff-only.

### Domain 2 — master data

- CRUD lokasi, kandang (kapasitas, strain, QR token), strain, grade telur, vendor.
- Penugasan staff ke kandang (`CageStaffAssignment`).
- `ProductionTarget` (HDP/FCR per `age_in_weeks`) — CRUD di halaman strain; belum dipakai di analytics.
- Siklus awal kandang (`CycleSetting`) saat pembuatan; manajemen siklus penuh belum.

### Domain 3 — operasional (fokus utama)

**Backend & admin**

- Migrasi `DailyProduction`: **TB / TR / TP** (bukan grade per baris); multi-record per kandang/hari.
- Rekap admin **Input harian** — layout tab (telur | pakan | populasi | pengobatan), bukan satu tabel raksasa.
- Tabel rekap telur: TB, TR, TP, Total, staff, waktu; kolom berat dihapus dari UI.
- Service + API: catat produksi, rekap harian, riwayat kandang, edit produksi.
- Unit test produksi (schema, parse tanggal, crack ratio, update) — 16 pass.

**Mobile**

- Navigasi cage-first: tab Input → hub → form / riwayat.
- **Form input harian tunggal** (accordion): Produksi, Konsumsi pakan, Mutasi populasi, Pengobatan.
- Produksi TB/TR/TP + total otomatis; intake pakan `(kg ÷ End layer) × 1000` g/ekor.
- Kematian hanya di mutasi populasi (bukan pengobatan).
- Riwayat kandang + edit per entri produksi (Opsi B multi-record).
- Antrean lokal (`AsyncStorage`) untuk pakan, populasi, pengobatan menunggu API.
- Scanner QR kamera end-to-end.

### Dokumen & referensi bisnis

- OpenAPI diperbarui untuk endpoint produksi & riwayat.
- Panduan strain Lohmann (`docs/Guidance untuk strain Lohman.md`) — acuan target HDP/FCR & program vaksinasi (belum diotomatisasi di sistem).
- Sample laporan harian klien — validasi format TB/TR/TP.
